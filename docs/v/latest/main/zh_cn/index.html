      向量常量

      向量常量

      返回数的图案有种神奇的性质：把向量之精思的尾部加到数之精思的开头，就能返回一个向量。

                  向量之精思，+1/-1第二型 (→ vector)

                      浏览器不支持图案的可视化。图案代码见此：qeaqaa 浏览器不支持图案的可视化。图案代码见此：qqdedd

                      向左微偏的头部会返回所有 3 个分量都相等的向量。


                      首先，绘制前页八种图案中的一种，默认数值为 0。然后，每一画的方向会以对应方式修改该数值。

                      前方：加 1

                      左前方：加 5

                      右前方：加 10

                      左后方：乘 2

                      右后方：除以 2
                      顺时针绘制，也即各页右图，会在绘制完成后取相反数（如左图逆时针绘制则不取）。

                      绘制完成后会返回对应向量。

                                                                                  保留式比较

                                                                                  总有些时候，我会想要在比较两个 iota 之后还继续对它们执行操作，但这就需要每次都先运行狄俄斯库里之策略，确实有些麻烦。


                                                                                  比较之策略

                                                                                  通过实验，我发现：在比较图案前加上一个顺时针的三角形，就可以保留两个输入参数了。


                                                                                  比较之策略：至大 (A, B → A, B, bool)

                                                                                  • 渲染方式
                                                                                    • 配色
                                                                                      浏览器不支持图案的可视化。图案代码见此：ddwe

                                                                                      如果第一个参数大于第二个，返回 True。否则返回 False。


                                                                                      比较之策略：至小 (A, B → A, B, bool)

                                                                                      • 渲染方式
                                                                                        • 配色
                                                                                          浏览器不支持图案的可视化。图案代码见此：ddeq

                                                                                          如果第一个参数小于第二个，返回 True。否则返回 False。


                                                                                          比较之策略：至大第二型 (A, B → A, B, bool)

                                                                                          • 渲染方式
                                                                                            • 配色
                                                                                              浏览器不支持图案的可视化。图案代码见此：ddwee

                                                                                              如果第一个参数大于或等于第二个，返回 True。否则返回 False。


                                                                                              比较之策略：至小第二型 (A, B → A, B, bool)

                                                                                              • 渲染方式
                                                                                                • 配色
                                                                                                  浏览器不支持图案的可视化。图案代码见此：ddeqq

                                                                                                  如果第一个参数小于或等于第二个，返回 True。否则返回 False。

                                                                                                                                          范围之提整 (num/vec, num, num, num → bool)

                                                                                                                                          • 渲染方式

                                                                                                                                              第四个数似乎描述了检验的方式。

                                                                                                                                              0：min < val < max

                                                                                                                                              1：min <= val < max

                                                                                                                                              2：min < val <= max

                                                                                                                                              3：min <= val <= max
